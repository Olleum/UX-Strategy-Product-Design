Контекст: Проработка пользовательского опыта и продуктовой логики сервиса для обеспечения быстрого освоения (принцип «3 кликов»).

User Story Mapping (USM): Разработала карту пользовательского пути для функционала «Семейный бюджет», разделив роли на «Родитель» (администратор, управление лимитами) и «Ребенок» (ограниченный доступ) 🗺️.

Feature Design:

Спроектировала механику установки лимитов на категории расходов с автоматическими уведомлениями при превышении 🔔.

Решила проблему «точек отказа»: добавила прозрачность периодов лимитов и визуализацию прогресс-баров на главном экране для повышения финансовой дисциплины пользователей 📈.

Prototyping: Создала Low-fidelity прототипы (4 экрана) в виде React-компонентов, включая интерфейсы настройки лимитов и аналитики расходов по категориям 💻.
